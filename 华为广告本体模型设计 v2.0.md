# 华为广告本体模型设计 v2.0

## 一、本体概述

本模型基于 Palantir Foundry 本体建模理念，通过 Object 类型（实体类型）、链接类型（关系）、操作类型和接口来构建华为广告系统的知识图谱，支持广告投放策略的自动化生成和优化。

## 三、链接类型（关系）

### 1. 用户相关关系

- **用户 → 使用 → 设备**：用户使用设备
  - 基数：一对多（一个用户可以使用多个设备，一个设备可以被多个用户使用）
  - 显示名称：使用的设备 / 使用该设备的用户
  - API名称：usedDevices / usersUsingDevice

- **用户 → 产生 → 行为事件**：用户产生行为事件（APP安装/卸载/打开/使用、浏览、搜索）
  - 基数：一对多（一个用户可以产生多个行为事件）
  - 显示名称：产生的行为事件 / 产生该行为事件的用户
  - API名称：userEvents / eventUser

- **用户 → 处于 → 用户场景**：用户处于特定场景
  - 基数：一对多（一个用户可以处于多个场景）
  - 显示名称：所处的场景 / 该场景中的用户
  - API名称：userScenarios / scenarioUsers

### 2. 应用相关关系

- **推广标的 → 属于 → 行业**：推广标的属于行业
  - 基数：多对一（多个推广标的可以属于一个行业）
  - 显示名称：所属行业 / 该行业的推广标的
  - API名称：industry / industryTargets

### 3. 广告投放相关关系

- **广告主 → 创建 → 广告计划**：广告主创建广告计划
  - 基数：一对多（一个广告主可以创建多个广告计划）
  - 显示名称：创建的广告计划 / 创建该计划的广告主
  - API名称：advertiserPlans / planAdvertiser

- **广告计划 → 包含 → 广告任务**：广告计划包含广告任务
  - 基数：一对多（一个广告计划可以包含多个广告任务）
  - 显示名称：包含的广告任务 / 所属广告计划
  - API名称：planTasks / taskPlan

- **广告任务 → 投放 → 媒体版位**：广告任务投放在媒体版位上
  - 基数：多对多（一个广告任务可以投放在多个媒体版位，一个媒体版位可以接受多个广告任务）
  - 显示名称：投放的媒体版位 / 接受该广告任务的媒体版位
  - API名称：taskPlacements / placementTasks

- **广告任务 → 推广 → 推广标的**：广告任务推广推广标的
  - 基数：多对一（多个广告任务可以推广一个推广标的）
  - 显示名称：推广的标的 / 接受该推广的广告任务
  - API名称：taskTargets / targetTasks

- **广告任务 → 使用 → 广告创意**：广告任务使用广告创意
  - 基数：多对一（多个广告任务可以使用一个广告创意）
  - 显示名称：使用的创意 / 被使用的广告任务
  - API名称：taskCreatives / creativeTasks

- **广告任务 → 采用 → 出价策略**：广告任务采用出价策略
  - 基数：多对一（多个广告任务可以采用一个出价策略）
  - 显示名称：采用的出价策略 / 被采用的广告任务
  - API名称：taskBiddingStrategies / biddingStrategyTasks

- **广告任务 → 目标 → 转化目标**：广告任务有转化目标
  - 基数：多对一（多个广告任务可以有同一个转化目标）
  - 显示名称：转化目标 / 有该转化目标的广告任务
  - API名称：taskConversionGoals / conversionGoalTasks

### 4. 行为相关关系

- **行为事件 → 关联 → 浏览内容**：行为事件关联浏览内容（仅适用于浏览行为）
  - 基数：多对一（多个行为事件可以关联同一个浏览内容）
  - 显示名称：关联的浏览内容 / 关联该内容的行为事件
  - API名称：eventBrowsingContent / browsingContentEvents

- **行为事件 → 关联 → 搜索内容**：行为事件关联搜索内容（仅适用于搜索行为）
  - 基数：多对一（多个行为事件可以关联同一个搜索内容）
  - 显示名称：关联的搜索内容 / 关联该内容的行为事件
  - API名称：eventSearchContent / searchContentEvents

### 5. 媒体流量相关关系

- **媒体 → 包含 → 流量位**：媒体包含流量位
  - 基数：一对多（一个媒体可以包含多个流量位）
  - 显示名称：包含的流量位 / 所属媒体
  - API名称：mediaAdSlots / adSlotMedia

- **流量位 → 属于 → 媒体版位**：流量位属于媒体版位（一个流量位可以属于多个媒体版位）
  - 基数：多对多（一个流量位可以属于多个媒体版位，一个媒体版位可以包含多个流量位）
  - 显示名称：所属媒体版位 / 包含的流量位
  - API名称：adSlotPlacements / placementAdSlots

### 6. 广告行为相关关系

- **广告请求 → 触发 → 广告竞价**：广告请求触发广告竞价
  - 基数：一对一（一个广告请求触发一个广告竞价）
  - 显示名称：触发的广告竞价 / 被触发的广告请求
  - API名称：requestBidding / biddingRequest

- **广告竞价 → 包含 → 召回阶段**：广告竞价包含召回阶段
  - 基数：一对一（一个广告竞价包含一个召回阶段）
  - 显示名称：包含的召回阶段 / 所属广告竞价
  - API名称：biddingRecall / recallBidding

- **广告竞价 → 包含 → 粗排阶段**：广告竞价包含粗排阶段
  - 基数：一对一（一个广告竞价包含一个粗排阶段）
  - 显示名称：包含的粗排阶段 / 所属广告竞价
  - API名称：biddingRoughSort / roughSortBidding

  - 基数：一对一（一个广告竞价包含一个精排阶段）
  - 显示名称：包含的精排阶段 / 所属广告竞价
  - API名称：biddingPreciseSort / preciseSortBidding

- **广告竞价 → 结果 → 广告曝光**：广告竞价成功后产生广告曝光
  - 基数：一对多（一个广告竞价可能产生多个广告曝光）
  - 显示名称：产生的广告曝光 / 对应的广告竞价

- **广告曝光 → 触发 → 广告点击**：广告曝光触发广告点击
  - 基数：一对多（一个广告曝光可能产生多个广告点击）
  - 显示名称：触发的广告点击 / 对应的广告曝光
  - API名称：exposureClicks / clickExposure

  - 基数：一对多（一个广告点击可能产生多个广告转化）

## 五、接口设计（基于 Palantir Foundry 概念）

### 1. 接口概述

接口是华为广告本体模型中的抽象类型，用于描述对象类型的形状及其功能，提供对象类型的多态性，允许对具有共同形状的对象类型进行一致的建模和交互。

### 2. 接口类型

#### 2.1 功能接口

##### 2.1.1 可推广对象接口（PromotableObject）
- **描述**：代表可以被推广的对象类型
- **共享属性**：对象ID、对象名称、对象类型、行业分类
- **实现对象类型**：推广标的

##### 2.1.2 可投放对象接口（DeployableObject）
- **描述**：代表可以被投放广告的对象类型
- **共享属性**：对象ID、对象名称、曝光价格、流量质量
- **实现对象类型**：媒体版位

#### 2.2 抽象对象接口

##### 2.2.1 用户行为接口（UserBehavior）
- **描述**：代表用户行为的抽象接口
- **共享属性**：行为ID、用户ID、行为时间、设备ID
- **实现对象类型**：APP安装行为、APP卸载行为、APP打开行为、APP使用行为、浏览行为、搜索行为

##### 2.2.2 投放场景接口（DeploymentScene）
- **描述**：代表广告投放场景的抽象接口
- **共享属性**：场景ID、场景名称、场景类型、适用行业
- **实现对象类型**：投放场景

## 六、数据覆盖说明

| 数据类型 | 覆盖率 | 处理方式 |
|----------|--------|----------|
| APP行为 | 较高 | 正常参与特征计算 |
| 用户画像 | 高 | 核心定向维度 |
| 搜索/浏览 | ~30% | 标记"未知"，作为补充信号 |

## 七、设计说明

### 1. 实体类型设计

- 核心实体：用户、设备、广告主、应用
- 上下文实体：用户场景
- 行为事件实体：APP安装、APP卸载、APP打开、APP使用、浏览行为、搜索行为、广告请求、广告曝光、广告点击、广告转化、广告竞价
- 内容实体：浏览内容、搜索内容
- 目标实体：转化目标
- 投放单元实体：广告计划、广告任务
- 推广对象实体：推广标的
- 内容创意实体：广告创意
- 策略规则实体：出价策略
- 分类实体：行业、汽车品牌
- 流量来源实体：媒体
- 广告位置实体：流量位
- 售卖单元实体：媒体版位
- 展示环境实体：投放场景
- 汽车相关实体：汽车、汽车用户
- 数据实体：投放数据
- 结果实体：指标

### 2. 关系设计

- 使用 Palantir Foundry 链接类型概念，明确实体之间的关系
- 关系类型包括：使用、安装、产生、处于、属于、创建、包含、投放、推广、使用、采用、目标、关联等
- 建立了用户行为与内容实体的关联关系，便于用户行为分析

### 3. 操作类型设计

- 明确用户和广告主可以执行的操作

### 4. 接口设计

- 提供查询和修改实体的接口
- 提供数据分析和投放优化的接口
- 接口设计便于系统集成和扩展

### 5. 数据处理

- 对覆盖率低的数据（如搜索/浏览行为）标记为"未知"，避免算法误判
- 对实体属性进行标准化处理，提高数据质量
